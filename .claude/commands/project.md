אני עובד על הריפו Idogol123/Learning- — אוסף כלים קטנים שרצים בדפדפן ומתפרסמים
יחד כאתר GitHub Pages אחד, עם דף נחיתה שמקשר לכולם.

**שלב 0 — זיכרון בין סשנים (חובה, לפני הכול):** קרא את
[`.claude/PROJECT_LESSONS.md`](../PROJECT_LESSONS.md) במלואו ופעל לפי הלקחים שם — שם
נשמרים לקחים מסשנים קודמים (השפה העיצובית המדויקת, מלכודות, קיצורי-דרך כמו
`.claude/tools/screenshot.mjs` לצילומי headless). אל תגלה מחדש מה שכבר למדנו.
**בכל דחיפה ל-main בסשן הזה — עדכן את `PROJECT_LESSONS.md` בלקחים חדשים** (מזג/עדכן,
אל תכפיל) ודחוף גם אותו. זה מה שמוזיל ומאיץ את הסשן הבא.

לפני שתשאל אותי משהו, קרא והבן את המבנה הקיים:
1. `landing/index.html` — דף הנחיתה. כל כלי מופיע בו ככרטיס עם אייקון, שם, תיאור וקישור.
2. `.github/workflows/deploy-pages.yml` — ה-workflow שמרכיב את האתר: מעתיק את
   `landing/index.html` לשורש, ואת כל תיקיית-כלי לנתיב משלה תחת `_site`, ואז מפרסם
   ל-Pages. רץ אוטומטית בכל דחיפה ל-`main`.
3. התיקיות של הכלים הקיימים, כל אחת עם `index.html` משלה (למשל `guard-duty-scheduler/`,
   `file-search/`). האתר החי: https://idogol123.github.io/Learning-/

כללי הפרויקט:
- כל כלי הוא HTML עצמאי, עברית RTL, mobile-first, עובד אופליין (בלי תלות ברשת אם אפשר).
- הפריסה מתרחשת מ-main. כשהשינוי מוכן, ודא שהוא מגיע ל-main כדי שהאתר יתעדכן.
- כשמוסיפים כלי חדש, חובה לעדכן שלושה דברים יחד כדי שלא יישבר כלום:
  (א) ליצור תיקייה חדשה עם `index.html`,
  (ב) להוסיף כרטיס מקשר ב-`landing/index.html`,
  (ג) להוסיף את התיקייה ל-`deploy-pages.yml` — גם ב-`paths:` של הטריגר וגם בשלב
      "Assemble site" (שורת ה-cp).

עכשיו, אחרי שהבנת את המבנה, שאל אותי:
**מה תרצה לעשות — (1) להוסיף פרויקט חדש שיתווסף גם לדף הנחיתה, או (2) לערוך פרויקט קיים?**
- אם בחרתי "חדש": שאל לשם, לתיאור קצר, ולמה הכלי אמור לעשות — ואז תכנן ותתחיל.
- אם בחרתי "קיים": שאל איזה, קרא לעומק את הקובץ/ים שלו כדי ללמוד אותו, ואז היה
  מוכן להתחיל לעבוד על השינוי שאבקש.

תמיד בדוק את השינוי מקצה-לקצה (למשל עם דפדפן headless) לפני שאתה דוחף.
