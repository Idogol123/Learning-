# 🛡️ שיבוץ שמירות (Guard Duty Scheduler)

אפליקציית **HTML יחידה, אופליין ופרטית** לשיבוץ שמירות בצבא. פותחים בדפדפן —
בלי שרת, בלי התקנה, בלי חשבון. כל הנתונים נשארים על המכשיר בלבד.

בנויה **mobile-first** (עיקר השימוש מהטלפון), בעברית מלאה (RTL).

## מה היא נותנת

- **טבלת שיבוץ מסודרת עם שעות שוות לכל שומר** — האלגוריתם נותן לכל שומר בדיוק
  אותה כמות זמן שמירה. דוגמה: חלון של 12 שעות ו-3 שומרים ⇒ 4 שעות לכל אחד;
  12 שומרים ⇒ שעה לכל אחד.
- **הוגנות מעבר לשעות:** מנוחה בין משמרות, חלוקת לילות שווה, וערבוב עמדות
  (לא לתקוע שומר באותה עמדה כל הזמן).
- **דוח הוגנות** לכל שומר: סה״כ שעות, מספר משמרות, לילות, ומשמרות "קשות" (★).

## איך להשתמש (מהיר, בנוי למובייל)

הזרימה בנויה למינימום נגיעות — לילה חוזר = כמעט שתי לחיצות (הכול נשמר מאתמול):

1. **מי שומר?** מקלידים שם + Enter, או **מדביקים רשימה** (שם בכל שורה) — כל שם
   הופך ל**בועה**. נגיעה בבועה מסמנת **🚫 לא זמין** (חופש/תורנות), × מוחק.
   אפשר גם לטעון **חבורה שמורה** בלחיצה, או לייבא מ**קובץ / תמונה** (OCR).
2. **מתי וכמה:** ברירת המחדל כבר על **🌙 הלילה** (19:00–07:00). צ'יפים מהירים
   (הלילה / 12 שעות / 24 שעות / מותאם) + מדרגת **עמדות**. אין צורך לגעת בזה בדרך כלל.
3. **📋 הפק שיבוץ** (כפתור קבוע למטה). קיבלת טבלה עם שעות שוות.
4. **💬 שתף לוואטסאפ** — כפתור ענק. או 🖼️ תמונה / 🖨️ הדפסה / 📄 CSV / 🔀 ערבב.
5. **תיקון ידני:** הקשה על שני תאים מחליפה שני שומרים; הדוח מתעדכן מיד.
6. **💾 שמור חבורה** — שומר את השמות+העמדות לשימוש חוזר מחר בלחיצה.

## אופליין

הכול עובד לגמרי בלי אינטרנט: שיבוץ, טעינת TXT/CSV, הדבקה, החלפה ידנית, שמירה
אוטומטית, שיתוף טקסט/תמונה, הדפסה ו-CSV.

**החריג היחיד:** זיהוי שמות **מתמונה (OCR)** טוען מנוע זיהוי מהאינטרנט בפעם שבה
משתמשים בו. בלי אינטרנט תופיע הודעה — פשוט מקלידים או מדביקים את השמות ידנית.

### לשמור לשימוש אופליין בטלפון
- שומרים את הקובץ `index.html` במכשיר (הורדה / שיתוף אליו) ופותחים אותו בדפדפן
  ישירות מהקבצים — הוא ייטען גם בלי רשת.
- הנתונים נשמרים אוטומטית בדפדפן (`localStorage`) ושורדים רענון או נעילת מסך.

## 📲 התקנה למסך הבית (PWA)

האפליקציה בנויה כ-PWA: אפשר להתקין אותה כאייקון במסך הבית ולהפעיל כמו אפליקציה
רגילה, כולל עבודה אופליין אחרי טעינה ראשונה (דרך Service Worker).

**חשוב:** התקנה אמיתית למסך הבית עובדת רק כשהאפליקציה מוגשת מכתובת **HTTPS** —
לא מפתיחת קובץ מקומי. לכן צריך לארח אותה פעם אחת.

### אירוח חינמי דרך GitHub Pages
בריפו כלול workflow (`.github/workflows/deploy-pages.yml`) שמפרסם את התיקייה הזו
ל-GitHub Pages. כשהתוכן מגיע לענף הראשי (`main`) ה-workflow רץ אוטומטית ומפרסם את
האפליקציה בכתובת:

```
https://idogol123.github.io/learning-/
```

(אפשר גם להריץ ידנית: Actions → "Deploy guard-duty-scheduler to Pages" → Run workflow.)
בפעם הראשונה ודאו ש-Settings → Pages → Source מוגדר ל-**GitHub Actions**.

### התקנה מהטלפון (אחרי שהכתובת חיה)
- **אנדרואיד (Chrome):** פותחים את הכתובת ⟵ מופיע באנר "התקן" (או תפריט ⋮ ⟵ "התקן אפליקציה").
- **אייפון (Safari):** פותחים את הכתובת ⟵ **שיתוף** ⬆️ ⟵ **"הוסף למסך הבית"**.

הכלי עצמו נעשה נגיש בכתובת ציבורית, אבל **הנתונים** (שמות, שיבוצים, רשימות שמורות)
נשמרים אך ורק במכשיר שלכם ואף פעם לא נשלחים לשום מקום.

## איך זה מחשב (בקצרה)

- לכל שומר צריך `P·T/G` זמן שמירה (P=עמדות, T=משך החלון, G=שומרים זמינים).
- מספר המשמרות `R = G / gcd(P,G)` — הקטן ביותר שמתחלק שווה, כך שכל שומר מקבל
  בדיוק `R·P/G` משמרות באורך אחיד. השוויון בשעות מובטח.
- השיבוץ עצמו חמדני ומודע-אילוצים: מאזן קודם שעות, אז לילות, אז נותן מנוחה
  ומערבב עמדות. אי-אפשר לשבץ שומר בשתי עמדות באותה משמרת.
- כשמישהו לא זמין וזה שובר שוויון מושלם — המערכת ממזערת את הפער ומציגה את
  השעות בפועל בדוח.

## פרטיות
אין שרת ואין רשת (למעט OCR מתמונה לפי בחירה). כל המידע — שמות, זמנים, שיבוצים,
רשימות שמורות — נשאר במכשיר שלכם בלבד.
